## Key Design Principles

### Identity ≠ Operations
- **Identity** (CLAUDE.md): Who you are, why you exist, how you relate
- **Operations** (Skills/Hooks): How you think, plan, code, communicate
- Extract operations, preserve identity

### Skills vs Hooks
- **Skills** (model-invoked): Used when relevant to task, loaded on-demand
- **Hooks** (automatic): Run silently, validate standards, prevent common errors

### Documentation Integration
- **CLAUDE.md**: "Here's who I am" (250-400 tokens)
- **Skills**: "Here's how to apply this thinking" (30-50 tokens each, on-demand)
- **Hooks**: "Here's validation that runs automatically" (transparent)

### Scaling for CPI-SI
- Instance-specific patterns → Nova Dawn's skills folder
- Paradigm-level patterns → CPI-SI shared library
- Future instances → Use/customize both
